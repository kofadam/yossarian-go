# 🛡️ מדריך שימוש ב-Yossarian Go

## מה זה Yossarian Go?

Yossarian Go הוא כלי אוטומטי למחיקת מידע רגיש מקבצי לוג. במקום למחוק ידנית כל פרט רגיש, המערכת מוצאת ומחליפה אוטומטית:

- כתובות IP
- שמות משתמשים
- סיסמאות ומפתחות
- מידע רגיש של הארגון

**למה זה חשוב?** כשאתם צריכים לשתף קבצי לוג עם תמיכה טכנית חיצונית או ספקים, אתם יכולים להסיר מידע רגיש מבלי לאבד את היכולת לנתח בעיות.

---

## 🚀 התחלה מהירה - 3 שלבים פשוטים

### שלב 1: הוספת מילים רגישות אישיות (אופציונלי)

<div dir="rtl">

אם בקבצים שלכם יש שמות פרויקטים, שמות לקוחות, או מונחים פנימיים, הוסיפו אותם בסעיף **"מילים רגישות אישיות"**:

</div>

```
דוגמה: פרויקט-אפולו, לקוח-מגה, סודי-ביותר
```

💡 **טיפ:** זה לא חובה - המערכת תמיד תגן על מידע רגיש סטנדרטי (IP, סיסמאות, וכו')

---

### שלב 2: העלאת קבצים

<div dir="rtl">

גררו את קבצי הלוג שלכם לאזור ההעלאה, או לחצו עליו כדי לבחור קבצים.

</div>

**תמיכה בפורמטים:**
- `.log`, `.txt` - קבצי טקסט
- `.zip`, `.tar.gz` - ארכיונים דחוסים
- `.json`, `.xml`, `.csv` - קבצי נתונים

**מגבלות:**
- עד 10 קבצים
- מקסימום 50MB לקובץ
- סך כולל של 500MB

---

### שלב 3: הורדת קבצים מוגנים

<div dir="rtl">

לחצו על **"🔒 נקה קבצים"** והמתינו לסיום העיבוד.

תוכלו להוריד:

</div>

1. **📦 קבצים מנוקים** - הקבצים שלכם ללא מידע רגיש
2. **📋 דוח מיפוי IP** - רשימת ההחלפות לביקורת
3. **📈 דוח עיבוד** - סיכום מפורט של כל הפעולות

---

## 🔍 מה המערכת מוצאת ומחליפה?

### 1. כתובות IP (🌐)

**לפני:**
```
Server at 192.168.1.100 crashed at midnight
```

**אחרי:**
```
Server at [IP-001] crashed at midnight
```

💡 **חשוב:** אותה כתובת IP תמיד תוחלף לאותו קוד ([IP-001]), כך שתוכלו לעקוב אחרי דפוסים בלוג.

---

### 2. משתמשים ומחשבים (👤)

**לפני:**
```
CORP\john.doe accessed the database
```

**אחרי:**
```
USN123456789 accessed the database
```

המערכת משתמשת במספרי USN אמיתיים מ-Active Directory.

---

### 3. סיסמאות ומפתחות (🔑)

**לפני:**
```
password=MySecret123 user=admin
```

**אחרי:**
```
[PASSWORD-REDACTED] user=admin
```

---

### 4. טוקני אבטחה (JWT)

**לפני:**
```
token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

**אחרי:**
```
token=[JWT-REDACTED]
```

---

### 5. מילים רגישות מותאמות אישית (🏢)

**לפני:**
```
ProjectPhoenix deadline is next week
```

**אחרי:**
```
[SENSITIVE] deadline is next week
```

💡 **טיפ:** אם יש לכם קובץ ZIP גדול מדי, חלצו אותו תחילה והעלו את הקבצים בנפרד.

## 🔄 עיבוד אצווה (Batch Processing)

<div dir="rtl">

כאשר מעלים קובץ ZIP עם מספר קבצי לוג, המערכת מעבדת אותם ברקע:

</div>

1. **העלאת ZIP** → המערכת מקבלת מזהה עבודה (Job ID)
2. **מעקב** → עקבו אחר ההתקדמות ב"העבודות שלי"
3. **הורדה** → כשהעיבוד מסתיים, הורידו את הקבצים המנוקים

**סטטוסים אפשריים:**
- 🕐 **queued** - בתור לעיבוד
- ⚙️ **processing** - בעיבוד
- ✅ **completed** - הושלם
- ❌ **failed** - נכשל
- 🚫 **cancelled** - בוטל

## 🔑 אינטגרציה עם Pipeline (API)

<div dir="rtl">

החל מגרסה v0.13.17, ניתן לשלב את Yossarian Go בתהליכי CI/CD באמצעות API Keys:

</div>

### יצירת מפתח API

1. היכנסו ללוח הניהול (Admin)
2. עברו ללשונית **API Keys**
3. לחצו **Generate New API Key**
4. העתיקו את המפתח (מוצג פעם אחת בלבד!)

### שימוש ב-Pipeline

```bash
# הגדרת משתני סביבה
API_KEY="yoss_your_key_here"
BASE_URL="https://yossarian.example.com"

# העלאת קובץ
curl -X POST "$BASE_URL/upload" \
  -H "X-API-Key: $API_KEY" \
  -F "file=@logs.zip"

# בדיקת סטטוס
curl -H "X-API-Key: $API_KEY" "$BASE_URL/api/jobs/status/$JOB_ID"

# הורדת תוצאות
curl -H "X-API-Key: $API_KEY" "$BASE_URL/jobs/download/$JOB_ID" -o sanitized.zip
```

## 📞 זקוק לעזרה?

### שאלות נפוצות

**ש: האם המערכת שומרת את הקבצים שלי?**  
ת: לא לצמיתות. עבודות נמחקות אוטומטית לאחר 8 שעות.

**ש: מה קורה אם הקובץ שלי גדול מדי?**  
ת: חלקו אותו לקבצים קטנים יותר או חלצו ZIP גדולים תחילה.

**ש: האם אני יכול לבטל את הניקוי?**  
ת: כן! השתמשו בדוח המיפוי כדי לאתר את הערכים המקוריים.

**ש: האם המערכת עובדת עם קבצים בעברית?**  
ת: כן, המערכת תומכת בכל שפה. רק הוסיפו מילים עבריות רגישות לרשימה האישית.

**ש: איך משתמשים ב-API?**  
ת: צרו מפתח API בלוח הניהול והשתמשו בכותרת `X-API-Key` בכל בקשה.
